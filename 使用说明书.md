# JND 主观实验 GUI 使用说明书

## 1. 软件用途

这个软件用于做 JND 主观实验。界面会顺序播放两段视频，让被试判断两段视频是否存在**可察觉的画面差异**，并自动保存训练结果、正式实验结果和最终安全配置集合。

## 2. 启动前准备

先在项目根目录安装依赖：

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

启动软件：

```bash
.venv/bin/python main.py
```

## 3. 数据目录要求

选择的 `Scene Folder` 必须是一个场景文件夹，路径最后三级应为：

```text
{device}/{action_type}/{scene_folder_name}
```

例如：

```text
/.../huaweipura/run/natlan_r01_run01
```

其中：

- `action_type` 要和场景名里的标签一致，例如 `run` 对应 `natlan_r01_run01`
- 场景文件夹里必须是 `.mp4` 视频
- 视频文件名支持两种格式：

```text
Resolution_FPS_Effect_Shadow.mp4
Resolution_Resolution_FPS_Effect_Shadow.mp4
```

- 必须包含参考视频 `VeryHigh + 60 FPS + High Effect + High Shadow`

## 4. 操作流程

### 4.1 开始实验

1. 打开软件。
2. 输入 `Subject ID`（受试者）。
3. 点击 `Browse Folder` 选择场景文件夹。
4. 点击 `Start Experiment`。

如果目录合法，界面会显示设备、动作类型、国家、路线编号、场景名和有效视频数量。

### 4.2 断点续做

如果同一个 `Subject ID` 和同一个场景已经有未完成会话，软件不会重建，而是进入 `Resume Existing Session` 页面。

- 点击 `Resume Session`：继续上次进度
- 点击 `Cancel`：返回首页重新选择

### 4.3 Training

每个被试者只需要完成一次训练。

- 第一次进入时，会进行 3 个训练 trial
- 如果该被试之前已经完成训练，软件会自动跳过训练，直接进入正式实验

每个 trial 的实际操作：

1. 点击 `Start Trial`
2. 播放 `Clip A`
3. 等待 5 秒
4. 播放 `Clip B`
5. 两段都结束后，点击：
   - `No noticeable difference`
   - `Visible difference`
6. 点击 `Start Next Trial` 进入下一题

训练结果不计入正式数据。

### 4.4 Formal Experiment

正式实验分两阶段：

- `Phase 1`：按分辨率逐步确定最低 JND-safe FPS
- `Phase 2`：只对 `Phase 1` 中找到安全 FPS 的分辨率继续测试 `Effect/Shadow` 组合

操作方式和训练一致：先播放 A、再等待 5 秒、再播放 B，最后选择是否有可察觉差异。

注意：

- 只有两段视频都播放完后，响应按钮才会启用
- 每次提交后要点击 `Start Next Trial` 才会进入下一题
- 程序会自动决定两段视频谁先播放，操作者不需要手动设置

## 5. 实验结果保存位置

每次正式实验的数据会保存到：

```text
Results/{subject_id}/{device}/{action_type}/{scene_folder_name}/
```

常见文件如下：

- `session_meta.json`：会话基础信息
- `session_state.json`：当前进度和恢复状态
- `raw_trials.jsonl`：每次正式 trial 的原始记录
- `phase1_result.json`：Phase 1 结果
- `phase2_result.json`：Phase 2 结果
- `final_jnd_safe_set.json`：最终 JND-safe 配置集合
- `candidate_power_prior_manifest.json`：候选配置清单

另外，训练完成状态会保存到：

```text
Results/{subject_id}/subject_training_state.json
```
